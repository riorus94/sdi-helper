OPERATOR REVIEW CHECKLIST
Side-View Wheel Detection Dataset — Phase 2
============================================

TIME PER IMAGE: 3–5 seconds.
DECISION RULE:  If you are unsure at first glance → press D (reject).
                When in doubt, discard.  Quality beats volume.

KEYBOARD CONTROLS (filter.py)
------------------------------
  a  — ACCEPT   → filtered/accepted/
  d  — REJECT   → filtered/rejected/
  r  — REVIEW   → filtered/review/   (revisit at end of session)
  q  — QUIT     (progress is auto-saved; safe to resume)

==============================================
ACCEPT if ALL seven conditions are true
==============================================

  [ ] 1. BOTH TYRES CLEARLY VISIBLE
         Front AND rear tyre are FULLY in frame — no tyre cut off at edge.
         No wheel hidden behind a body panel, object, or crop edge.

  [ ] 2. PURE SIDE VIEW
         Camera is perpendicular to the vehicle's long axis (≤10° yaw).
         You can see the full flank but NOT the front bumper face.

  [ ] 3. ENTIRE VEHICLE IN FRAME
         Hood, trunk, and roof are not cut off.
         (Partial crop of mirrors is OK.)

  [ ] 4. CLEAN / NEUTRAL BACKGROUND
         White, light-grey, or studio-style background.
         NO street scenes, buildings, parking lots, or busy backgrounds.
         NO outdoor scenery (trees, roads, traffic).

  [ ] 5. NO SHADOWS
         No hard shadow cast under or beside the vehicle.
         No strong directional sunlight creating shadow patterns.
         Studio-lit or evenly-lit images only.

  [ ] 6. REAL PHOTOGRAPH ONLY
         Must be a real photograph of a real car.
         NOT: illustrations, cartoons, vector art, CGI renders, toy cars.
         Watermarks are acceptable if they do not cover the wheels.

  [ ] 7. CORRECT VEHICLE TYPE
         Passenger car only: sedan, SUV, hatchback, estate.
         NOT: motorcycles, pickup trucks, buses, vans, sports prototypes.

==============================================
REJECT IMMEDIATELY if ANY of the following
==============================================

  [X] One tyre is cropped, occluded, or missing from frame
  [X] Angle is clearly diagonal (3/4 front or rear quarter)
  [X] Frontal or rear view
  [X] Street / outdoor / parking lot background
  [X] Visible shadow under the vehicle
  [X] Cartoon, illustration, or vector art
  [X] Heavy motion blur
  [X] Screenshot with UI overlays or logos covering the wheel area
  [X] Truck, van, bus, motorcycle, or racing prototype
  [X] Rendering / CGI only image (no photo-realistic detail)
  [X] Extreme crop (only door panel visible, no wheel present)

==============================================
REVIEW (r key) — acceptable borderline cases
==============================================

  • Slight yaw (5–10°) but both tyres clearly visible
  • Minor watermark in top corner (not over wheels)
  • One tyre very slightly cut at frame edge but still annotatable
  • Vehicle partially behind another object but both wheels clear

  Revisit REVIEW images at the end of each collection session.
  Apply same ACCEPT/REJECT rules with fresh eyes.

==============================================
ANNOTATION READINESS TARGET
==============================================

  Goal             : ≥ 1 200 accepted images before annotation starts
  Train / val split: 80 / 20  (automatic via split_dataset.py)
  Min to start     : 50 accepted images (quick sanity run)
  mAP50 target     : ≥ 0.70  (val set)
  Recall target    : ≥ 0.80

  At 4 sec/image: 1 200 images ≈ 80 minutes of operator time.
  Split across 2–3 sessions to maintain decision quality.

LEGAL / ETHICAL NOTES
==============================================

  • Do NOT redistribute the raw image dataset.
  • Do NOT publish the dataset to any public repository or cloud bucket.
  • Keep urls.csv (source attribution) alongside the images at all times.
  • Images from Bing Image Search may be copyrighted.
    They are used under fair-use / research exemption for non-commercial
    internal ML development only.

    with explicitly licensed or self-photographed images before release.

==============================================
POST-REVIEW PIPELINE
==============================================

  1. python collection/collect.py          # download candidates
  2. python collection/filter.py           # operator manual review
  3. python collection/split_dataset.py    # copy accepted → dataset/images/train+val
  4. Annotate with LabelImg or Roboflow    # bbox format (see dataset/README.txt)
  5. python yolo_training/train.py         # Phase 1 training
  6. Copy best.pt → cv_service/models/     # deploy
  7. Set USE_YOLO=1 and smoke-test         # validate
